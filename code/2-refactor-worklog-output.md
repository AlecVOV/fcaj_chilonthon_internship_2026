## Impact Analysis: LaTeX → Markdown Migration

### 1. Files That NEED to Change

#### useReportExport.ts — **Major rewrite**
This is the **core file**. Currently it:
- Embeds a full `LATEX_TEMPLATE` string with `\documentclass`, `\usepackage`, `\begin{tabularx}`, etc.
- Has `sanitizeForLatex()` with Vietnamese NFD normalization, LaTeX escape chars
- Uses `{{PLACEHOLDER}}` template syntax
- Generates a `.tex` file for download

**Change to:** Replace the LaTeX template with a **Markdown template string** using the same `{{PLACEHOLDER}}` syntax. Remove `sanitizeForLatex()` — Markdown doesn't need LaTeX special character escaping (just basic HTML entity encoding if needed).

```markdown
# Daily Focus Report — {{REPORT-DATE-FULL}}

**User:** {{USER-DISPLAY-NAME}}
**Email:** {{USER-EMAIL}}

## Summary
| Metric | Value |
|--------|-------|
| Total Focus Time | {{TOTAL-FOCUS-TIME}} min |
| Sessions | {{SESSIONS-COUNT}} |
| Tasks Completed | {{TASKS-COMPLETED}} / {{TASKS-TOTAL}} |
| Dominant Mood | {{DOMINANT-EMOTION}} |
| Streak | {{STREAK-DAYS}} days |

## Tasks
| Status | Task | Duration (min) |
|--------|------|---------------|
{{TASK-LIST-ROWS}}

## Insights
- **Peak Focus Hour:** {{PEAK-FOCUS-HOUR}}
- **Most Productive Day:** {{MOST-PRODUCTIVE-DAY}}

> "{{MOOD-SUMMARY-TEXT}}"

## AI Suggestions
{{AI-SUGGESTIONS-BLOCK}}

Generated by Focus Mode App — {{APP-URL}}
```

#### latex-template.tex — **Delete or archive**

### 3. Downstream Linkages (How Features Connect)

```
Dashboard "Export Report" button
    │
    ▼
ExportReportButton.vue
    │ calls useReportExport().exportReport()
    ▼
useReportExport.ts
    │
    ├─► Collects data: tasks, sessions, emotions, user info
    │
    ├─► Renders template string (LaTeX → change to Markdown)
    │
    ├─► Lambda path: POST /api/report → Lambda compiles PDF → returns S3 URL
    │   (Lambda would need to use Pandoc instead of Tectonic)
    │
    └─► Fallback: Browser downloads .md file (was .tex)
```
